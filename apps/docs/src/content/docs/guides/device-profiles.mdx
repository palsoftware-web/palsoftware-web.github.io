## Hardware focus

Pastiera is not a general soft keyboard first. It is aimed at Android devices with a real physical keyboard, and some behavior is profile-aware.

## Current baseline devices

The app currently carries device-specific assets for:

- Titan 2
- Q25
- Key2

Titan 2 has historically been the main reference device in the project. Q25 support is newer and should still be considered a more recent addition.

## What device support changes

Depending on the hardware, Pastiera may adapt:

- key remapping
- alternate key behavior
- gesture comfort
- status-bar usage expectations
- how natural Nav Mode or swipe interactions feel

The app is still fundamentally one product, but not every feature behaves equally well on every keyboard shape.

## Q25 status

Q25 support is present in the current app assets and codebase. That is a meaningful change from older docs, where Q25 was not documented at all.

The safest way to think about Q25 support today is:

- supported
- usable
- still worth verifying carefully after each major input-behavior change

## Accessibility and platform limits

Pastiera also exposes accessibility-related settings in `Settings > Accessibility`, including how the second row is announced and how quickly suggestions are read.

Some limits are not project bugs but Android or OEM constraints. That especially applies to:

- clipboard access behavior
- background restrictions
- certain pre-unlock or security-sensitive scenarios

## Recommended validation after setup

If you are configuring Pastiera on a new device, test these areas first:

1. normal typing
2. long press and modifier state changes
3. Nav Mode
4. SYM pages and emoji picker
5. clipboard history
6. backup and restore
