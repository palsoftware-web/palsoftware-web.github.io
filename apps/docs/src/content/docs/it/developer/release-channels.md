## Stable versus nightly

Pastiera distingue attualmente fra stable e nightly.

- stable e il canale pubblico normale
- nightly serve per verifiche piu rapide e build in anteprima

## Superfici nightly

Le nightly build sono oggi pubblicate o referenziate tramite:

- release nightly su GitHub
- repository F-Droid nightly ospitato via Pages

## Comportamento degli aggiornamenti

Il controllo aggiornamenti dipende dal build flavor e dal canale. Alcune build interrogano GitHub direttamente, altre orientate a F-Droid no.

## Firma e catena di fiducia

Le attestazioni di firma e gli altri materiali di trust appartengono al contesto operativo upstream e restano volutamente fuori dal percorso principale della documentazione funzionale.

## Nota per maintainer

Quando cambia il comportamento di rilascio, aggiorna questa pagina insieme alle pagine interne di progetto.
